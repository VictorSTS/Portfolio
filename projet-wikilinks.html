Description du projet

Jeu multijoueur compétitif où les joueurs s'affrontent dans une course à travers Wikipédia, en naviguant uniquement via les hyperliens pour atteindre un article cible. Le jeu met à l'épreuve la culture générale et la capacité de raisonnement des joueurs.

Technologies utilisées

React.js MongoDB Node.js API Wikipédia Git

Compétences mobilisées

Réaliser — Réaliser un développement d'application

Niveau 1 — Développer des applications informatiques simples

Création de l'interface de jeu de base avec React : affichage des articles Wikipédia, navigation par hyperliens, compteur de clics.

Niveau 2 — Partir des exigences et aller jusqu'à une application complète

Développement de l'application complète avec système de lobby multijoueur, synchronisation en temps réel, classement et historique des parties.

Niveau 3 — Adapter des applications sur un ensemble de supports

Application React responsive avec interface adaptée au mobile, gestion des interactions tactiles pour la navigation entre articles.

Collaborer — Travailler dans une équipe informatique

Niveau 1 — Identifier ses aptitudes pour travailler dans une équipe

Évaluation de mes compétences en React et en intégration d'APIs pour contribuer efficacement au développement front-end du jeu.

Niveau 2 — Situer son rôle et ses missions au sein d'une équipe informatique

Responsabilité du développement de l'interface utilisateur et de l'intégration de l'API Wikipédia, coordination avec l'équipe back-end.

Niveau 3 — Manager une équipe informatique

Animation des réunions de sprint, organisation du workflow Git (branches feature, pull requests, code review) pour l'ensemble de l'équipe.

Optimiser — Optimiser des applications informatiques

Niveau 1 — Appréhender et construire des algorithmes

Conception de l'algorithme de sélection aléatoire des articles de départ et d'arrivée avec vérification qu'un chemin existe entre eux.

Niveau 2 — Sélectionner les algorithmes adéquats pour répondre à un problème donné

Utilisation d'algorithmes de parcours de graphe (BFS) pour valider l'existence d'un chemin entre deux articles Wikipédia avant le début d'une partie.

Niveau 3 — Analyser et optimiser des applications

Mise en cache des articles Wikipédia consultés pour réduire les appels API, optimisation du rendu React avec mémoïsation des composants lourds.
